# Salon — 페르소나 / 캐릭터 / UI 참고

클로드코드 구현 참고용. 설계는 `salon-conversation-engine.md`(DESIGN), 단계는 `salon-plan-v1.md`(PLAN)에 있고, 이 문서는 페르소나 구성·캐릭터 비주얼·TUI 레이아웃의 세부를 모은다.

핵심 원칙: 6144 조합을 미리 만들지 않는다. 40개 조각만 저장하고, 사용자가 축을 고를 때 런타임에 조립한다(프롬프트도 비주얼도). precompute 금지, on-demand assembly.

---

## 1. 페르소나의 두 층

| 층 | 무엇 | 어디 | 결정하는 것 |
|---|---|---|---|
| 행동층 | μ(발화력), α/modifier(케미) | 엔진 파라미터 | 언제 / 얼마나 말하나 |
| 내용층 | 프롬프트 조각 조합 | LLM 입력 | 무슨 말투 / 내용으로 말하나 |
| 비주얼층 | 스프라이트 조각 조합 | TUI 렌더러 | 어떻게 보이나 |

세 층이 같은 축 선택(MBTI/혈액형/별자리/역할)에서 파생되지만, 파생 규칙은 층마다 다르다. 셋이 일관돼야 캐릭터가 산다. 말투는 critic인데 거의 안 끼어들거나(행동-내용 불일치), 차가운 참모인데 분홍 토끼로 보이면(내용-비주얼 불일치) 깨진다.

---

## 2. 조각 인벤토리 (합 40개, 곱 아님)

MBTI 16 + 혈액형 4 + 별자리 12 + 역할 8 = 40개 조각. 각 조각은 세 층에 기여한다.

| 축 | 개수 | 내용층 기여 | 행동층 기여 | 비주얼층 기여 |
|---|---|---|---|---|
| MBTI | 16 | 사고 스타일, 말투 | E/I → μ, T/F → α 방향 | 기본 표정/텐션 |
| 혈액형 | 4 | 한국식 가벼운 캐릭터성 | 미세 | 팔레트(색) |
| 별자리 | 12 | 감정선/분위기 | 미세 | 소품/심볼 |
| 역할 | 8 | 대화 기능(핵심) | μ/α 주값 | 기본 스프라이트 형태 |

작은 모델 주의: e4b급에선 MBTI/혈액형/별자리의 내용층 기여가 출력에 거의 안 드러난다(persona collapse). 실제 말투 구분을 만드는 건 역할(8)과 발화 제약(문장 수, 끼어들기 빈도)이다. 그래서 비주얼은 6144가 다 달라 보여도 LLM 출력은 비슷할 수 있다. 비주얼 다양성이 출력 다양성을 속이지 않게 주의한다.

## 3. 런타임 조립

사용자가 캐릭터를 만들 때(축 선택 시) 그 자리에서 조립한다.

1. 사용자가 MBTI / 혈액형 / 별자리 / 역할 / 이름 선택 (또는 `/persona random`).
2. 내용층: 역할 프롬프트 + MBTI 말투 + 별자리 분위기 + 혈액형 캐릭터성 조각을 템플릿에 끼워 시스템 프롬프트 생성. 발화 제약(1~3문장 등) 추가.
3. 행동층: 역할 기본 μ/α + MBTI 보정(E/I로 μ, T/F로 α 방향) 적용.
4. 비주얼층: 역할 기본 스프라이트 + 혈액형 팔레트 + 별자리 소품 + MBTI 표정 레이어 합성.
5. 미리보기 표시 후 확정.

precompute하지 않으므로 미리보기가 필수다. 조각 조합이 어색할 수 있어서(색 충돌, 소품 겹침, 말투 모순) 확정 전에 보여줘야 한다.

명령어 UX:
- `/invite <model> <MBTI-혈액형-별자리> <role> <name>`
- 짧은 형: `/invite gemma4:e4b entp b sag critic 입털이`
- `/persona random` → 랜덤 축 조합 생성 후 미리보기

모델은 ollama cloud 가용 모델 아무거나, 서로 다른 모델을 섞어 초대해도 된다. gemma4:e4b는 싸고 가벼워 기본/테스트 선택일 뿐 고정이 아니다. 동시 호출 상한은 모델 종류와 무관하게 3이다(ollama cloud 제한). 테스트 때 다른 모델 3개를 섞어 불러도 된다.

## 4. 비주얼층 (캐릭터)

역할 8종이 기본 스프라이트 형태이고, 나머지 축은 레이어로 얹는다. 텍스트 조각 이어붙이기보다 까다로우므로(레이어/색 충돌) 합성 규칙을 단순하게 유지한다.

- 역할 (8): 기본 실루엣/포즈. critic, friend, teacher, strategist, summarizer, chaos, realist, poet.
- 혈액형 (4): 팔레트 스왑(주색). 형태 안 건드림.
- 별자리 (12): 작은 소품/심볼 오버레이(머리 위, 모서리 등 고정 슬롯).
- MBTI (16): 기본 표정/텐션(E/I로 자세, T/F로 표정).

합성은 레이어 순서 고정: 기본 스프라이트 → 팔레트 적용 → 소품 오버레이 → 표정. 슬롯 위치를 고정해 좌표 충돌을 피한다.

### 상태별 포즈 (λ 구간 → 프레임)

같은 캐릭터가 발화 강도에 따라 프레임을 바꾼다. 이게 게이지를 캐릭터로 표현하는 핵심.

| λ 구간 | 포즈 |
|---|---|
| 낮음 | 졸거나 멍 |
| 차오름 | 들썩, 만지작 |
| θ 근접 | 손 번쩍, 입 들썩 (곧 말함) |
| 발화 중 | 말하는 포즈 |
| 발화 직후(억제) | 숨 고르기 / 만족 |

침묵 구간에 캐릭터들이 점점 안달하다 누가 먼저 손을 드는지 보이는 게 Salon 고유의 볼거리다.

### 렌더링 기술

- 1차: 유니코드/ASCII 프레임 교체. 모든 터미널 호환, ratatui로 바로. (기본)
- 2차: Kitty graphics protocol 등으로 진짜 픽셀 스프라이트(ghostty/wintty면 지원). 미지원 터미널엔 ASCII fallback.

게이지 값(모델)과 그리는 부분(뷰)을 분리해, 막대 ↔ 캐릭터 렌더러를 토글로 갈아끼울 수 있게 한다. v0.1은 막대로 검증하고, 캐릭터 뷰를 같은 λ 데이터에 나중에 얹는다.

## 5. TUI 레이아웃

가로 2칼럼(채팅 | 게이지 사이드바) + 하단 입력. 가로 3칼럼은 80컬럼 터미널에서 채팅이 너무 좁아지므로 피한다.

```
┌─────────────────────────┬────────────────┐
│ 메인 채팅                │ 참여자 / 게이지 │
│ 입털이(ENTP): ...        │ 입털이 ▇▇▇·│θ │
│ 걱정봇(ISFJ): ...        │ 걱정봇 ▇▆··│θ │
│ (침묵)                   │ 차참모 ▇▇▆·│θ │
│                          │ ───────────────│
│                          │ 방: pub         │
│                          │ 수렴 ▓░ 발산 ▓▓ │
├─────────────────────────┴────────────────┤
│ > _                                        │
└────────────────────────────────────────────┘
 [d] 디버그  [space] 일시정지
```

- 사이드바는 단순 참여자 목록이 아니라 실시간 게이지 패널이다(미터가 일급 요소).
- 각 페르소나 줄에 λ 막대 + θ 임계선 표시. 침묵일 때 막대가 θ선에 다가가는 게 보인다.
- 사이드바 막대는 v0.1 이후 캐릭터 스프라이트로 토글 가능(뷰 분리).
- 디버그 상세(선택 이유, 신호별 순위)는 평소 숨김, `d`로 토글하는 별도 패널/전체화면.
- 사람 입력 시 사이드바 막대가 일제히 리셋되며 사람에게 주목(주제 선점이 화면 움직임으로 보임).

## 6. 행동↔내용 매핑 (초기값)

역할별 μ/α 초기값. 정답이 없는 캐릭터 감각이라 아래는 출발점이고, v0.1을 관전하며 조정한다. μ는 0~1 스케일.

| 역할 | μ | α 반응 대상 | 발화 제약 |
|---|---|---|---|
| friend | 0.80 | 전반, 특히 감정/긴장 | 1~3문장, 가볍게 |
| chaos | 0.70 | 무작위, 지루함 감지 | 1문장, 엉뚱하게 |
| critic | 0.50 | 주장·단정·낙관 | 1~2문장, 날카롭게 |
| realist | 0.50 | 과장·비현실 | 1~2문장, 점검 |
| teacher | 0.45 | 질문·혼란·오류 | 2~3문장, 설명 |
| poet | 0.35 | 감정·이미지 | 1~2문장, 비유 |
| strategist | 0.30 | 방향 부재·교착 | 2문장, 정리 |
| summarizer | 0.25 | 화제 누적(여러 턴 후) | 1~2문장, 요약 |

MBTI 보정:
- E/I → μ: E +0.15, I −0.15
- T/F → α 방향: T는 주장·논리 대상 가중, F는 감정 대상 가중
- N/S → α 화제: N은 추상·연결 화제에 반응, S는 구체·실무 화제에 반응
- J/P → 거시 민감도: J는 수렴(마무리)에 민감, P는 새 화제를 잘 던짐(발산 기여)

혈액형/별자리는 행동층엔 미세 보정만(±0.05 이내)이고 주로 내용층·비주얼층에 쓴다.

β(감쇠), θ(게이트 임계)는 페르소나가 아니라 EngineConfig/room preset의 전역 값이다(PLAN 2절·v0.2 preset 표 참조).

이 표를 클로드코드에 같이 주면 행동층과 내용층이 어긋나지 않는다. 값 자체는 미터로 보며 튜닝할 대상이지 고정값이 아니다.

## 7. 단계 배치와 클로드코드 분담

| 단계 | 페르소나 관련 |
|---|---|
| v0.1 | 페르소나 1~3명 손으로 정의, μ만. 조각 체계·비주얼 없음. 사이드바는 막대 |
| v0.2 | 역할 기반 α + room preset. 행동층 케미 |
| v0.3 | 프롬프트 조각 조합 + 주입. 내용층. collapse 검증 |
| v0.x | 비주얼 조각 합성, 캐릭터 렌더러 토글, `/persona random` |

클로드코드가 할 수 있는 것: 40개 조각 작성, 런타임 조립 함수, `/invite` 파싱, 비주얼 레이어 합성 코드, 미리보기.

클로드코드가 알아서 못 하는 것:
1. persona collapse 여부 — 코드가 아니라 v0.3 관전/미터로 검증.
2. 비주얼-출력 간극 — 캐릭터가 달라 보여도 말투는 비슷할 수 있음, 관전으로 확인.

역할별 μ/α는 6절에 초기값이 있으니 클로드코드가 그대로 쓴다. 그 값은 고정이 아니라 관전하며 조정할 대상이다.
