주제 하나 던져놓고 LLM 페르소나들끼리 티격태격하게 만드는 브라우저 토론 앱입니다. 법정 공방이든 민초 논쟁이든, 페르소나들은 서로 기억하고, 반박하고, 닉네임을 불러가며 싸웁니다. 그리고 분위기가 달아오를 때쯤 당신이 슬쩍 끼어들면 다들 우르르 반응합니다.

근데 진짜 주인공은 페르소나가 아닙니다. tunaSalon의 본체는 **누가 언제 입을 열고, 언제 다 같이 조용해지는지**를 정하는 대화 흐름 엔진입니다. 페르소나는 거기에 얹힌 목소리일 뿐이에요. (원래는 그 엔진만 구경하는 터미널 장난감이었습니다. 지금도 `cargo run` 한 줄이면 그 시절 미터를 볼 수 있습니다.)

> 말을 시키는 건 쉽습니다. 어려운 건 자연스럽게 입을 다물게 만드는 쪽이에요. 이 프로젝트는 일부러 거꾸로 갑니다: 침묵부터 설계합니다.

## 보통 페르소나 채팅과 뭐가 다른가

흔한 "LLM에 페르소나 입혀서 대화" 방식은 대개 이렇습니다: 순서대로 답하거나, 부른 쪽이 답하거나, 매 턴 전원이 한마디씩. 타이밍도, 침묵도, "이번엔 그냥 듣고 있을게"도 없습니다. 끼어들 줄 모르는 패널 토론이죠.

tunaSalon에서는 페르소나마다 **발언 욕구**가 시간에 따라 차오르고 가라앉습니다. 아무도 욕구가 충분히 높지 않으면 방은 조용해집니다. 둘이 동시에 말하고 싶어하면 한 명만 골라줍니다. 대화는 알아서 달아오르고 알아서 식습니다.

대화 규칙을 일일이 짜는 게 아닙니다. 손잡이 몇 개를 돌리고 **구경**하면, 성격이 리듬으로 떨어져 나옵니다: 수다쟁이, 과묵한 애, 가끔만 끼어드는 애.

## 엔진 원리를 아주 짧게

작은 부품 네 개입니다.

| 손잡이 | 의미 |
|---|---|
| `μ` 말수 | 페르소나의 기본 수다력 (0~1) |
| `λ` 발언 압력 | 매 틱 μ를 향해 회복하고, 말하고 나면 뚝 떨어짐 (그래서 아무도 독점 못 함) |
| `θ` 침묵 문턱 | 매 틱 어느 λ도 이 선을 못 넘으면 방은 침묵. 높을수록 조용해짐 |
| `RRF` 화자 선택 | 여럿이 문턱을 넘으면 세 신호를 합쳐 한 명 선택: 누가 제일 말하고 싶은지(λ) + 최근 덜 말했는지(공정성) + 약간의 무작위 |

매 틱 도는 루프는 이렇습니다.

```
λ를 μ 쪽으로 살짝 당김  ->  게이트: 아무도 θ 못 넘으면 침묵
                       ->  넘은 후보가 있으면 RRF로 한 명 선택
                       ->  그 화자의 λ를 떨굼  ->  반복
```

여기까지는 LLM이 필요 없습니다. 가짜 한 단어 발언만으로도 리듬이 보입니다. 성격은 **타이밍**에 삽니다.

### 손잡이를 돌리면 (실측)

`cargo run -- --sweep`은 같은 seed에서 θ를 바꿔봅니다. μ는 friend 0.80, chaos 0.70, summarizer 0.25 고정:

```
θ=0.40  침묵   0   friend 100  chaos 100  summarizer 0   # 문턱 헐거움 -> 전원 통과, 쉴 새 없는 핑퐁
θ=0.65  침묵 100   friend  62  chaos  38  summarizer 0   # 문턱이 물어 -> 말/침묵 리듬, μ가 드러남
θ=0.78  침묵 171   friend  29  chaos   0  summarizer 0   # 문턱 빡빡 -> 제일 수다스러운 애만 겨우
```

μ는 그대로인데 θ 하나로 방이 논스톱에서, 리듬으로, 거의 침묵으로 바뀝니다.

### 케미 (α)

v0.2는 **교차 자극 α**를 더합니다: 한 페르소나가 말하면 다른 애들의 욕구가 올라갑니다. 누가 누구를 들쑤시는지가 곧 방의 케미고, 프리셋으로 분위기를 고릅니다.

```
preset=Calm      침묵 99   friend 67  chaos 34  summarizer  0   # 약한 α -> 조용한 애는 계속 조용
preset=Argument  침묵  0   friend 76  chaos 76  summarizer 48   # 강한 α -> 교차 자극이 조용한 애를 끌어냄
```

같은 summarizer(μ=0.25)가 Calm에선 한 번도 안 말하고 Argument에선 48번 끼어듭니다. 페르소나는 동일한데, **방 분위기가 과묵한 애를 대화로 끌어낸** 거예요.

### TUI 미터

`cargo run`이면 TUI가 뜹니다. λ 막대가 θ 선(`|`)을 향해 오르내리고, 누가 왜 말했는지가 실시간으로 보입니다.

```
┌events──────────────────────────────────────┐┌gauges────────────────────┐
│t8 (silence)                                ││Chaos Guest               │
│t9 (silence)                                ││########|.... 0.63        │
│t10 Friendly Regular                        ││Friendly Regular          │
│                                            ││########|.... 0.67        │
│                                            ││Quiet Summarizer          │
│                                            ││###.....|.... 0.25        │
│                                            ││speak 11  silence 6       │
└────────────────────────────────────────────┘└──────────────────────────┘
```

창발 시스템은 미터 없이 못 고칩니다. 방이 왜 조용해졌는지, 왜 한 페르소나가 독점하는지를 숫자로 보면서 튜닝합니다.

## 브라우저 토론 앱

엔진 트랙(v0.1~v0.10)은 늘 *수단*이었습니다: 생동감. 제품은 방이고, 그 방이 이제 브라우저에서 여는 **주제 기반 LLM 토론 앱**이 됐습니다 (axum WebSocket + React, `--features "web redis-bus"`).

유능한 에세이만 주고받는 토론은 지루합니다. 재미는 **연출자(producer)** 레이어에서 나오고, 갓파일을 안 키우려고 독립 모듈 `src/debate/`에 따로 삽니다.

- **DebatePlan** - 주제를 결정적으로(LLM 메타콜 없이) 토론 **모드**에 매핑합니다: `Courtroom`, `PolicyDuel`, `MoralDilemma`, `PersonalStakes`, `Forecasting`, `DesignReview`, 그리고 민초/부먹찍먹 같은 가벼운 주제용 `CasualBanter`. "AI 판사가 공정할까?"는 법정, "AI 규제와 오픈소스"는 정책 다툼이 됩니다. 모드가 방을 붙잡아 주제에서 안 새게 합니다.
- **형식 변주** - 매 턴이 같은 길이가 아니라 형식이 돌아갑니다: 교차신문, 스틸맨 후 반박, 구체 사례 하나, 측정 가능한 임계값, 조건부 양보. 일부는 일부러 한두 문장이라 발언이 일률적이지 않습니다.
- **루프 차단 twist 카드** - 방이 수렴(FlowMeter)하거나 같은 화자가 반복하면, 새 국면(오판 통계, 규제 부담에 떠난 유지보수자)을 투입해 반응을 강제합니다.
- **친구끼리 가볍게** - 진지한 주제여도 톤은 "법정처럼 딱딱하게 말고 친구끼리 따지듯". 분석 초점(근거·전제·트레이드오프)은 살리되 레지스터만 낮춥니다.

전부 생성 스냅샷에만 주입되고 저장 히스토리는 안 건드려서, LLM-off 골든 경로는 바이트 동일하게 유지됩니다.

**방에서 만날 수 있는 것:**

- **성격 있는 페르소나** - 혈액형/MBTI/별자리 4축으로 조립됩니다. 이게 *누구인지*(성향 서술)뿐 아니라 *어떻게 말하는지*(말투 디렉티브)까지 정합니다. ENTP는 "근데 그건-" 하고 끼어들어 역질문하고, A형은 조심스럽게 완곡하게, 물고기자리는 비유로 풀어냅니다. 같은 모델이어도 갈라집니다.
- **두 가지 방 생성** - "만들기"는 room id 기반 **결정적 랜덤 3명**(같은 주제 -> 같은 셋), "직접 고르기"는 4축으로 **2~3명을 직접 구성**하며 입장 전 닉네임을 미리 봅니다.
- **당신은 1급 참여자** - 끼어들면 전 페르소나의 욕구가 확 오르고, 몇 턴 동안 당신 말이 최우선 화제가 됩니다. 닉네임을 부르면 그 페르소나가 다음에 답하고요. 끝난 토론에 한마디 던지면 다시 살아납니다.
- **종료 리포트** - 토론이 닫히면 중립 분석가가 두괄식 마크다운 리포트를 씁니다: 결론, 참가자 전원의 입장(당신 것도), 합의점, 끝까지 갈린 지점.
- **읽기전용 공유** - 재밌게 끝난 토론은 토큰 링크로 외부에 공유할 수 있습니다. 받은 사람은 로그인 없이 전사(아바타·색상·결론까지)를 PC/모바일에서 읽습니다. 공유 누른 방만 공개되고요.
- **이전 토론 아카이브** - 지나간 방은 다 보관됩니다. 시작/결론 날짜와 함께 목록에서 골라 다시 들어가면 참가자·주제·히스토리가 복원됩니다.
- **발언 게이지** - 참가자별 Hawkes 욕구 λ가 θ 선에 대한 막대로 뜹니다. 넘으면 발화 *자격*이 생기고(한 명은 RRF가 선택), 차오름-발화-억제-침묵 리듬이 그대로 보입니다.

**Redis 멀티세션**(`redis-bus` feature, `SALON_REDIS_URL`로 opt-in)은 *휘발성 코디네이션* 레이어입니다: 방 command/event 스트림, owner lease, presence. 기억 저장소가 아니에요. `LiveSession`이 방당 단일 writer로 남고 SQLite(`memory.db`, `rooms.db`)가 영속 SSOT입니다. Redis를 비워도 히스토리는 안 사라집니다.

제품 트랙 전체가 `web` / `redis-bus` feature 뒤에 있습니다. 기본 빌드와 headless 골든 출력은 무손상입니다.

## 실행하기

[Rust](https://rustup.rs)만 있으면 됩니다. 기본 실행은 LLM도 네트워크도 필요 없습니다.

```bash
cargo run                                         # TUI 미터 보기. q 종료, space 일시정지
cargo run -- --chat                               # 방에 참여: 인터랙티브 채팅 (실제 터미널 필요)
cargo run --features "web redis-bus" -- --web --topic "AI 판사가 공정할까?"  # 브라우저 토론 앱
cargo run -- --headless --ticks 200               # 틱당 한 줄 결정적 NDJSON
cargo run -- --sweep                              # θ × k 격자 + 프리셋 비교
cargo run -- --room argument                      # 방 분위기 프리셋 (calm/pub/argument/chaos)
cargo run -- --room chaos --fsm                   # 케미 + 같은 화자 2연속 금지
cargo run -- --theta 0.7 --k 5 --beta 0.4         # 손잡이 직접 조정
cargo run -- --llm                                # LLM opt-in (기본 클라우드 모델, 네트워크 필요)
cargo run --features friend-engine -- --chat      # 세션 넘어 기억 (형태소 + SQLite BM25 회상)
cargo run --features friend-engine-semantic -- --chat  # + 의미 회상 (모델 있으면 BGE-M3 로드)
cargo run --example persona_collapse              # 같은 모델, 두 페르소나 비교 (Ollama 필요)
cargo run --example mixed_bench                   # 클라우드 + 지인 vLLM 혼합 (두 백엔드 필요)
cargo run --example chat_demo                     # 흐름 수렴 수치가 줄마다 찍히는 비대화형 채팅
cargo test                                        # 306 tests (friend-engine 319 / semantic 347 / web 356)
```

| 옵션 | 의미 |
|---|---|
| `--theta` | 침묵 문턱 |
| `--k` | RRF 1등 쏠림 완화 |
| `--beta` | 발언 압력 회복/감쇠 속도 |
| `--room` | 방 분위기 프리셋 |

같은 `--seed`면 출력은 매번 바이트 동일합니다. 그래서 headless로 자동 검증됩니다.

## 왜 만들었나

LLM 몇 개를 한 주제에 붙여놓으면 매 턴 전부 답합니다. 영원히. 끼어들지도, 참지도, 지루해하지도 않는 패널이죠. 논쟁은 그렇게 안 굴러갑니다. 진짜 논쟁엔 *타이밍*이 있어요: 누군가 끼어들고, 누군가는 입을 다물고, 방은 같은 지점을 맴돌다 누군가 새 각도를 끌고 들어옵니다. 그 어느 것도 프롬프트에 없습니다. 리듬입니다.

그래서 tunaSalon은 리듬을 먼저 만듭니다: 누구의 욕구가 차오르는지, 누가 방금 말했는지, 방이 언제 조용해져도 되는지. 그리고 거기서 캐릭터와 싸움이 떨어져 나오게 둡니다. 모델은 말만 채워요. 흥미로운 건 누가 그 말을 할지, 그리고 언제 침묵을 그대로 둘지 정하는 쪽입니다.
