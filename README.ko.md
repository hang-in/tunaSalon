# tunaSalon

![Rust](https://img.shields.io/badge/Rust-2021-CE422B?logo=rust&logoColor=white)
![status](https://img.shields.io/badge/status-v0.2-blue)
![tests](https://img.shields.io/badge/tests-50%20passing-brightgreen)
![no LLM yet](https://img.shields.io/badge/v0.1--v0.2-no%20LLM%20yet-8A2BE2)
![determinism](https://img.shields.io/badge/output-deterministic-informational)

로컬 LLM 페르소나들을 터미널 대화방에 초대해 잡담시키는 TUI 앱입니다.

다만 핵심은 페르소나 자체가 아닙니다.
tunaSalon의 본체는 누가 언제 말하고, 언제 조용해지고, 어떤 분위기에서 대화가 달아오르는지를 만드는 대화 흐름 엔진입니다.

보통 멀티에이전트 시스템은 문제를 풀기 위해 돌아갑니다.
하지만 잡담은 다릅니다. 정답도 없고, 명확한 종료 조건도 없습니다. 누군가는 끼어들고, 누군가는 듣고만 있고, 대화는 잠깐 달아올랐다가 다시 식습니다.

tunaSalon은 그 흐름을 만들기 위한 실험입니다.

LLM에게 “다음은 누가 말할까?”를 묻지 않습니다.
화자 선택과 침묵은 가벼운 알고리즘이 담당하고, LLM은 실제 발언 내용만 생성합니다.

«말하게 하는 것보다, 자연스럽게 조용해지게 만드는 쪽이 더 어렵습니다.
tunaSalon은 그 이상한 문제를 다루는 프로젝트입니다.»

---

보통 페르소나 채팅과 뭐가 다른가

일반적인 페르소나 채팅은 대개 단순합니다.

- 순서대로 대답한다
- 사용자가 부른 쪽이 답한다
- 모든 페르소나가 매번 한마디씩 한다

이 방식에는 대화의 리듬이 없습니다.
말할 타이밍도 없고, 끼어들기 감각도 없고, “이번엔 그냥 듣고 있겠다”는 선택도 없습니다.

tunaSalon은 각 페르소나마다 발언 게이지를 계산합니다.

발언 게이지는 시간이 지나면 조금씩 차오르고, 말을 하고 나면 내려갑니다. 누군가의 말에 자극받으면 다시 올라갑니다. 아무도 충분히 말할 상태가 아니면 방은 조용해집니다. 여러 명이 동시에 말할 수 있는 상태가 되면, 엔진이 그중 한 명만 고릅니다.

그래서 대화는 항상 이어지지 않습니다.

어떤 방은 쉴 새 없이 떠들고,
어떤 방은 몇 마디 하다 조용해지고,
어떤 방은 한 페르소나가 계속 끼어들고,
어떤 방은 조용하던 페르소나가 특정 분위기에서 갑자기 말하기 시작합니다.

성격은 대사에만 있는 게 아니라 타이밍에도 있습니다.

---

지금 구현된 것

현재 버전은 LLM 없이도 대화 리듬이 살아나는지를 검증하는 단계입니다.

실제 모델이 말을 생성하기 전에, 먼저 다음을 확인합니다.

- 누가 말할 상태가 되는가
- 아무도 말하지 않는 침묵이 생기는가
- 방 분위기에 따라 조용한 페르소나가 끌려 나오는가
- 특정 페르소나가 대화를 독점하지 않는가
- 같은 설정에서 항상 같은 결과가 나오는가

즉 v0.1~v0.2의 목표는 “똑똑한 말”이 아니라 그럴듯한 대화 흐름입니다.

---

알고리즘을 아주 짧게 설명하면

tunaSalon의 엔진은 네 가지 손잡이로 움직입니다.

손잡이| 의미
"μ" 말수| 페르소나의 기본 수다력
"λ" 발언 압력| 지금 이 페르소나가 얼마나 말할 차례에 가까운지
"θ" 침묵 문턱| 이 선을 못 넘으면 아무도 말하지 않음
"RRF" 화자 선택| 말할 후보가 여럿일 때 최종 화자를 고르는 방식

매 틱마다 엔진은 이렇게 움직입니다.

각 페르소나의 발언 압력을 갱신한다
→ 아무도 침묵 문턱을 넘지 못하면 조용히 있는다
→ 후보가 있으면 한 명을 고른다
→ 그 페르소나가 말한 것으로 기록한다
→ 발언 압력과 방 분위기를 다시 갱신한다

여기까지는 LLM이 필요 없습니다.
가짜 발언만으로도 “누가 언제 말하는지”의 리듬을 볼 수 있습니다.

---

손잡이를 돌리면 무엇이 달라지나

"cargo run -- --sweep"으로 같은 조건에서 침묵 문턱 "θ"를 바꿔보면 방 분위기가 달라집니다.

말수는 다음과 같이 고정했습니다.

friend     μ = 0.80
chaos      μ = 0.70
summarizer μ = 0.25

결과는 다음과 같습니다.

θ=0.40  침묵   0   friend 100  chaos 100  summarizer 0
θ=0.65  침묵 100   friend  62  chaos  38  summarizer 0
θ=0.78  침묵 171   friend  29  chaos   0  summarizer 0

문턱이 낮으면 방이 쉬지 않고 떠듭니다.
문턱이 적당하면 말과 침묵이 섞인 리듬이 생깁니다.
문턱이 높으면 가장 수다스러운 페르소나만 겨우 말합니다.

같은 페르소나라도, 침묵 문턱 하나로 방의 리듬이 완전히 달라집니다.

---

케미 - 누가 누구를 자극하는가

v0.2에서는 교차 자극 "α"가 들어갑니다.

한 페르소나의 발언이 다른 페르소나의 발언 압력을 끌어올립니다.
이 값이 곧 방의 케미입니다.

예를 들어 같은 페르소나 구성을 두고 방 분위기만 바꾸면 결과가 달라집니다.

preset=Calm      침묵 99   friend 67  chaos 34  summarizer  0
preset=Argument  침묵  0   friend 76  chaos 76  summarizer 48

Calm 방에서는 조용한 summarizer가 한 번도 말하지 않습니다.
Argument 방에서는 교차 자극이 커져서 summarizer도 48번이나 끼어듭니다.

페르소나는 그대로인데, 방 분위기가 조용한 쪽을 대화로 끌어낸 것입니다.

TUI 미터

"tunaSalon"은 대화 흐름을 숫자로 관찰합니다.

┌events──────────────────────────────────────┐┌gauges────────────────────┐
│t8 (silence)                                ││Chaos Guest               │
│t9 (silence)                                ││########|.... 0.63        │
│t10 Friendly Regular                        ││########|.... 0.67        │
│                                            ││Quiet Summarizer          │
│                                            ││###.....|.... 0.25        │
│                                            ││speak 11  silence 6       │
└────────────────────────────────────────────┘└──────────────────────────┘

막대 안의 "|"가 침묵 문턱입니다.
막대가 그 선을 넘은 페르소나만 말할 수 있습니다.

하단의 "reason"은 이번 틱에서 왜 그 페르소나가 선택됐는지를 보여줍니다.

- 발언 압력이 가장 높았는지
- 최근 덜 말해서 균형상 선택됐는지
- 무작위 신호가 작동했는지

창발 시스템은 미터 없이 디버깅할 수 없습니다.
tunaSalon은 대화가 왜 조용해졌는지, 왜 특정 페르소나가 독점하는지 숫자로 보면서 튜닝합니다.

실행하기

Rust만 있으면 됩니다.
v0.1~v0.2는 LLM도 네트워크도 필요 없습니다.

cargo run                                    # TUI 미터 보기
cargo run -- --headless --ticks 200          # 틱당 한 줄 NDJSON 출력
cargo run -- --sweep                         # 파라미터 스윕
cargo run -- --room argument                 # 방 분위기 프리셋
cargo run -- --room chaos --fsm              # 케미 + 같은 사람 2연속 금지
cargo run -- --theta 0.7 --k 5 --beta 0.4    # 손잡이 직접 조정
cargo test                                   # 테스트 실행

주요 손잡이는 다음과 같습니다.

옵션| 의미
"--theta"| 침묵 문턱
"--k"| RRF에서 1등 쏠림을 완화하는 값
"--beta"| 발언 압력이 회복/감쇠되는 속도
"--room"| 방 분위기 프리셋

같은 "--seed"를 쓰면 출력은 매번 같습니다.
그래서 헤드리스 모드로 자동 검증할 수 있습니다.

현재 상태

v0.1 - 리듬

말수, 침묵 문턱, 화자 선택만으로 발언과 침묵의 리듬을 검증했습니다.

v0.2 - 케미

교차 자극 "α"를 추가했습니다.
누가 누구의 발언 압력을 끌어올리는지, 방 분위기 프리셋에 따라 대화가 어떻게 달라지는지 확인합니다.

현재 구현:

- Hawkes 기반 발언 압력
- 침묵 문턱
- RRF 화자 선택
- 교차 자극 "α"
- 방 프리셋 "calm / pub / argument / chaos"
- FSM 전이 옵션
- TUI 미터
- 결정적 headless 출력
- 테스트와 스모크 게이트

v0.3 - 로컬 LLM

다음 단계에서는 Ollama를 붙입니다.

화자 선택은 tunaSalon 엔진이 담당하고, 실제 대사 생성만 LLM이 맡습니다.
목표는 작은 로컬 모델이 서로 다른 페르소나를 유지할 수 있는지 확인하는 것입니다.

특히 볼 것은 "persona collapse"입니다.

«같은 작은 모델에 다른 페르소나를 주입했을 때,
시간이 지나며 말투와 관점이 비슷하게 무너지는가?»

---

왜 만들었나

멀티에이전트 LLM 데모는 대부분 과제를 풀기 위해 돌아갑니다.

문제를 나누고, 역할을 주고, 답을 만들고, 종료합니다.
그건 회의나 작업에는 맞습니다.

하지만 잡담은 다릅니다.

잡담은 답을 향해 수렴하지 않습니다.
목표도 없고, 승인 키워드도 없고, 명확한 종료 조건도 없습니다.
그냥 누군가 말하고, 누군가는 듣고, 대화가 잠깐 달아올랐다가 식습니다.

tunaSalon은 그 흐름을 다루는 실험입니다.

대사를 잘 만드는 것보다,
말할 타이밍과 침묵을 만드는 쪽이 핵심입니다.

---

한 줄 정의

tunaSalon은 로컬 LLM 페르소나를 위한 TUI 대화 흐름 엔진입니다.

페르소나는 부품이고, 본체는 다음 네 가지입니다.

누가 말할 상태가 되는가
누가 실제로 말하는가
언제 아무도 말하지 않는가
대화가 어떻게 달아올랐다 식는가

자세한 설계는 ""docs/reference/salon-engine-design.md"" (docs/reference/salon-engine-design.md)를 참고하세요.
