로컬 LLM 페르소나들을 한 방에 풀어놓고 잡담시키는 터미널 앱. 
단, 진짜 주인공은 페르소나가 아니라 **누가 언제 말하고 언제 입을 다무는지**를 정하는 대화 흐름 엔진입니다.

에이전트들이 말을 하게 하는 건 쉽습니다. 하지만 침묵을 설계하는 게 어렵습니다. 좀 거꾸로 된 프로젝트입니다.

---

## 보통 페르소나 채팅과 뭐가 다른가

흔한 "LLM에게 페르소나를 부여하고 채팅"하는 방식은 매 턴 비슷합니다. 

> **차례대로 대답하거나, 호명된 쪽이 답하거나.** 

말 할 타이밍 감각도, 침묵도, "지금은 그냥 듣고 있겠다" 같은 결정도 없습니다.

tunaSalon은 각 페르소나가 **발언 욕구**를 가집니다. 이 욕구가 시간에 따라 차오르고 식습니다. 누구의 욕구도 충분히 높지 않으면 채팅은 조용해 집니다. 그리고 둘이 동시에 말하고 싶어지면 누가 낄지를 정해 한 명만 뽑습니다. 대화가 알아서 달아올랐다 식었다가 합니다.

단순히 대사 규칙을 짜 넣은 게 아닙니다. 대화 몇 턴을 돌리고 **지켜보면** 성격이 리듬처럼 보입니다. 수다스러운 쪽, 조용한 쪽, 가끔만 끼어드는 쪽.

## 알고리즘, 쉽게

작은 부품 네 개면 됩니다. 각 부품에는 수학 기호가 하나씩 붙습니다.

1. **말수(μ)** - 페르소나마다 정하는 기본 발화 성향입니다. 0~1 사이 값입니다.
2. **발화 욕구(λ)** - 매 틱 말수 쪽으로 회복하고, 방금 말한 쪽은 뚝 떨어집니다(독점 방지). 한 번의 발화가 다음 발화를 부르고 시간이 지나면 잦아드는 self-exciting point process, 곧 Hawkes 과정의 단순화판입니다.
3. **침묵 문턱(θ)** - 매 틱 어떤 발화 욕구도 이 문턱을 못 넘으면 방은 조용해집니다. 문턱을 높일수록 침묵이 잦아집니다.
4. **화자 선택(RRF)** - 문턱을 넘은 후보가 여럿이면 신호 셋을 섞어 한 명을 뽑습니다. 누가 제일 말하고 싶은가(발화 욕구), 최근에 누가 적게 말했는가(공정성), 그리고 약간의 무작위. 점수 스케일이 달라도 순위만으로 합치는 Reciprocal Rank Fusion 방식입니다.

매 틱 루프는 이게 전부입니다.

```
모든 발화 욕구를 말수 쪽으로 살짝 회복
  → 침묵 문턱: 아무도 못 넘으면 침묵
  → 넘었으면 화자 선택으로 한 명
  → 그 사람 발화 욕구를 떨어뜨림
  → 반복
```

리듬을 만드는 데 LLM은 필요 없습니다. 가짜 한마디만으로도 타이밍이 돕니다. 성격은 **타이밍**에 들어 있습니다.

### 손잡이가 뭘 하는가 (실제 출력)

`cargo run -- --sweep`으로 같은 시드에서 침묵 문턱(θ)과 동점 처리(k)를 훑은 결과입니다. 말수(μ)는 friend 0.80, chaos 0.70, summarizer 0.25.

```
θ=0.40  침묵   0   friend 100  chaos 100  summarizer 0   # 문턱이 헐거우면 다 통과, 쉴 새 없는 핑퐁
θ=0.65  침묵 100   friend  62  chaos  38  summarizer 0   # 문턱이 물리면 발화와 침묵의 리듬, 말수 차이가 드러납니다
θ=0.78  침묵 171   friend  29  chaos   0  summarizer 0   # 문턱이 빡빡하면 제일 수다스러운 쪽만 겨우 한마디
```

같은 말수인데 침묵 문턱 하나로 "쉴 새 없는 방 / 리듬 있는 방 / 거의 조용한 방"이 갈립니다. (참고로 발화 분산은 동점 처리(k)보다 공정성 신호가 좌우합니다. 그래서 위 표에서 k를 바꿔도 분포가 거의 안 변합니다. 이런 걸 미터로 보면서 알아갑니다.)

### 케미 (v0.2)

v0.2는 **교차 자극(α)**을 더합니다. 한 페르소나의 발화가 다른 페르소나의 발화 욕구를 끌어올립니다. 누가 누구에게 발끈하는지가 곧 방의 케미입니다. 방 분위기는 프리셋으로 고릅니다.

```
preset=Calm      침묵 99   friend 67  chaos 34  summarizer  0   # 자극 약함. 조용한 쪽은 계속 조용
preset=Argument  침묵  0   friend 76  chaos 76  summarizer 48   # 자극 강함. 교차 자극이 조용한 쪽까지 끌어들입니다
```

같은 summarizer(말수 0.25)가 Calm에서는 한 번도 안 말하고, Argument에서는 48번 말합니다. 페르소나는 그대로인데 **방 분위기가 조용한 쪽을 대화로 끌어들이는** 셈입니다.

### 미터

`cargo run`으로 TUI를 띄우면 발화 욕구 막대가 침묵 문턱 선(`|`)에 다가갔다 멀어지고, 누가 왜 말했는지가 실시간으로 보입니다.

```
┌events──────────────────────────────────────┐┌gauges────────────────────┐
│t8 (silence)                                ││Chaos Guest               │
│t9 (silence)                                ││########|.... 0.63        │
│t10 Friendly Regular                        ││Friendly Regular          │
│                                            ││########|.... 0.67        │
│                                            ││Quiet Summarizer          │
│                                            ││###.....|.... 0.25        │
│                                            ││speak 11  silence 6       │
└────────────────────────────────────────────┘└──────────────────────────┘
┌──────────────────────────────────────────────────────────────────────────┐
│tick 10 | len 17 | Friendly Regular | reason: intensity   [q] quit [space]  │
└──────────────────────────────────────────────────────────────────────────┘
```

막대 안의 `|`가 침묵 문턱입니다. 막대가 그 선을 넘은 페르소나만 말할 자격이 생깁니다. `reason`은 이번에 그 페르소나가 뽑힌 이유입니다(발화 욕구가 가장 높았는지, 공정성 차례였는지).

## 해보기

[Rust](https://rustup.rs)만 있으면 됩니다. (v0.1~v0.2는 LLM도 네트워크도 필요 없습니다.)

```bash
cargo run                                    # 미터를 라이브로 보기 (TUI). q 종료, space 일시정지
cargo run -- --headless --ticks 200          # 결정적 NDJSON, 틱당 한 줄
cargo run -- --sweep                         # 침묵 문턱 × 동점 처리 격자 + 프리셋 비교
cargo run -- --room argument                 # 방 분위기 프리셋 (calm/pub/argument/chaos)
cargo run -- --room chaos --fsm              # 케미 + 같은 사람 2연속 금지
cargo run -- --theta 0.7 --k 5 --beta 0.4    # 손잡이 돌려보기
cargo test                                   # 50개 테스트 (스모크 게이트 포함)
```

손잡이는 넷입니다.

- **말수(μ)** - 페르소나별 기본 발화 성향
- **침묵 문턱(θ)** - 방이 조용해지는 기준선
- **동점 처리(k)** - 화자 선택에서 동점을 가르는 정도
- **회복 속도(β)** - 발화 욕구가 다시 차오르는 속도

같은 `--seed`면 출력은 매번 똑같습니다(결정적). 그래서 헤드리스로 자동 검증이 됩니다.

## 왜

멀티에이전트 LLM 데모는 거의 다 과제를 풀고 끝납니다. 답이 나오면 종료. 그런데 진짜 잡담엔 과제도 종료 조건도 없습니다. 그냥 흐르다 식습니다. 그건 다른 종류의 엔진이 필요했고, 그래서 찔러볼 수 있는 걸 하나 만들었습니다.
